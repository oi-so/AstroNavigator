# 04. Planetarium

## 1. 目的

プラネタリウムは、本ソフトウェアの中心機能である。

天球上の天体を正確かつ高速に表示し、観測・導入・追尾・撮影支援の基盤となることを目的とする。

---

## 2. 基本方針

表示は「天球」を基準とする。

すべての天体は共通の座標系を経由して描画する。

描画方法は天体の種類によって変えない。

## 14. SkyObject

プラネタリウム上に表示されるすべての要素は `SkyObject` を基底とする。

例：

- 恒星
- 惑星
- 月
- 太陽
- 彗星
- 人工衛星
- 架台
- カメラ画角
- Plate Solve結果

各SkyObjectは現在時刻・観測地点などに応じた位置情報を提供する共通インターフェースを持つ。


## 15. Layer

描画要素はレイヤー単位で管理する。

各レイヤーは独立して

- 表示／非表示
- 描画順
- 透明度（将来）
- 設定

を持つ。

例：

- 恒星
- 惑星
- DSO
- 人工衛星
- 軌道
- グリッド
- 天の川
- ラベル
- 架台
- カメラ画角


## 16. 観測モード

観測モードはレイヤー構成のプリセットとして実装する。

例：

- 観望モード
- 撮影モード
- 人工衛星モード
- ISS追尾モード

ユーザーは独自の観測モードを作成・保存できるものとする。


## 17. Scene

プラネタリウムは `Scene` を単位として描画する。

Scene は以下の要素から構成される。

- Observer（観測地点）
- Time（時刻）
- SkyCamera（視点）
- Projection（投影法）
- LayerManager（表示レイヤー）
- SkyObjects（表示対象）

Renderer は Scene を入力として描画を行う。
