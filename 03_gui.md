# 03. GUI Design

## 1. 設計方針

GUIは「観測支援」を第一目的とする。

プラネタリウムを中心に、観測・導入・追尾に必要な情報へ素早くアクセスできる構成とする。

GUIは初心者から上級者まで利用できるよう設計する。

---

## 2. レイアウト

GUIは Dock Layout を採用する。

各パネルは

- ドッキング
- 分離
- サイズ変更
- 非表示

を可能とする。

標準レイアウトでは中央にプラネタリウムを配置する。

```
+--------------------------------------------------------------+
| Menu                                                         |
+--------------------------------------------------------------+
| Toolbar                                                      |
+--------------------------------------------------------------+
| Object |                     Sky                     | Mount  |
| Search |                                             | Camera |
|--------|                                             | Sat    |
| Plan   |                                             | Log    |
+--------+---------------------------------------------+--------+
| Status Bar                                                   |
+--------------------------------------------------------------+
```

3. 表示モード

対応

* ダーク（標準）
* ライト
* ナイトモード（赤画面）

⸻

4. プラネタリウム

標準は2D表示とする。

将来的には3D風表示へ切り替え可能な設計とする。

レンダリング部は表示方式から独立させる。

⸻

5. 表示内容

表示可能

* 恒星
* 星座
* 星座線
* 天の川
* 惑星
* 月
* 太陽
* 彗星
* 人工衛星
* 軌道
* 地平線
* グリッド
* 架台位置
* 観測経路
* カメラ画角（将来）

各表示はON/OFF可能とする。

⸻

6. ラベル

ラベルは

* ズーム倍率
* 画面密度
* 優先順位

に応じて自動表示する。

重なりは可能な限り避ける。

表示対象はユーザーが設定可能とする。

⸻

7. レンダリング

レンダリングエンジンはGUIから独立させる。

将来的な変更を考慮し、Rendererインターフェースを設ける。

初期実装では、開発効率と性能のバランスを考慮した描画方式を採用する。

⸻

8. 更新

描画更新は約60fpsを目標とする。

描画速度は処理負荷に応じて変動可能とする。

描画よりも

1. ISS追尾
2. ユーザー操作
3. 描画

の順に優先する。

⸻

9. 情報表示

表示例

* RA
* Dec
* Alt
* Az
* LST
* UTC
* JST
* FPS
* マウス座標
* 架台座標

⸻

10. モード

GUIは用途に応じて切り替え可能とする。

シンプルモード

初心者・観望会向け。

必要最小限の機能のみ表示する。

プロモード

観測・撮影・ISS追尾など全機能を利用できる。

⸻

11. 優先事項

GUIでは特に以下を見つけやすい位置へ配置する。

* 天体検索
* オブジェクト選択
* GoTo
* 接続状態
* 導入
* Tracking
* ISS追尾
* 観測状態

これらは常に素早く操作できることを目標とする。
